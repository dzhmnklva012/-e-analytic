# ADATA Compliance — Модуль «Статистика»

Страница статистики для системы **ADATA Compliance**, построенная на
**Next.js 16 + React 19 + Tailwind v4 + shadcn/ui + Recharts + Lucide**.
Шрифт — **Open Sans**. Тема — светлая (clean shadcn).

> Отдельный, самостоятельный проект. Не использует и не затрагивает HTML-макеты
> в корне репозитория (`main.html`, `anticor.html`, `ionyx.html`, `login.html`).

## Запуск

Node.js установлен как portable-сборка в
`%LOCALAPPDATA%\node-portable\node-v24.16.0-win-x64` (нет в системном PATH).

```bash
# из пап. compliance-statistics
npm install      # зависимости уже установлены
npm run dev      # http://localhost:3300  (см. .claude/launch.json)
npm run build    # production-сборка
```

Если `node`/`npm` не находятся в PowerShell, добавьте portable-папку в PATH сессии:

```powershell
$env:PATH = 'C:\Users\Avrora\AppData\Local\node-portable\node-v24.16.0-win-x64;' + $env:PATH
```

## Данные на странице

1. **Задачи** — запланированные, завершённые, в работе, ожидают, прочие (донат).
2. **Файлы по компаниям** — вкладки «Проверки / Заключения / Данные», для каждой
   всего файлов + подписано / не подписано.
3. **Файлы по сотрудникам** — вкладки «Данные по месяцам» (помесячный график
   подписано/не подписано), «Проверки», «Заключения».
4. **Горячая линия** — всего, новые, в работе, завершённые.
5. **Расследования** — всего, новые, в работе, завершённые.
6. **Конфликт интересов** — всего, выявлен, не выявлен (донат).
7. **Подарки** — декларации, подарки, дарители, получатели.

Фильтры: **Компания** (Users → Companies) и **Период**
(сегодня / неделя / месяц / квартал / год / произвольный).

## Состояния

Все обязательные состояния реализованы и доступны через переключатель
**«Демо-состояние»** в панели инструментов или по deep-link:

| Состояние           | Как открыть       | Что показывает                       |
| ------------------- | ----------------- | ------------------------------------ |
| Данные              | `/`               | Полная статистика                    |
| Загрузка (skeleton) | `/?state=loading` | Скелетоны во всю сетку               |
| Пусто               | `/?state=empty`   | Пустые состояния в каждой секции     |
| Ошибка              | `/?state=error`   | Блок ошибки с кнопкой «Повторить»    |
| Офлайн              | `/?state=offline` | Баннер офлайна + последние данные    |
| Валидация           | `/?period=custom` | Произвольный период с проверкой дат  |

Офлайн также определяется реально через `navigator.onLine`
(`useOnlineStatus`) — баннер появляется при потере сети.

## Принципы реализации

- **Переиспользуемые компоненты** — секции собраны из небольших примитивов
  (`MetricTile`, `DonutStat`, `SignedSplitBar`, `SectionCard` …).
- **Сетка отступов и текста 4px** — размеры шрифта 12 / 16 / 20 / 24 / 28px,
  отступы и зазоры кратны 4. Минимальный размер текста — 12px.
- **Доступность с клавиатуры** — компоненты на shadcn/Base UI, skip-link,
  `aria-current`, `role="region"` для секций, ошибки валидации через
  `role="alert"` + `aria-invalid` + `aria-describedby`, графики с `role="img"`
  и текстовой подписью.
- **Адаптивность** — сайдбар сворачивается в Sheet, сетки 2 кол. на мобильном
  и до 3–5 на десктопе.
- **Скорость** — анимации графиков отключены, серверная отрисовка статической
  страницы, имитация fetch с отменой по `AbortController`.
